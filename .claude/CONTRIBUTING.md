# 貢献ガイド

EcorisMapのClaude Code設定への貢献方法です。

## 📋 貢献の種類

### 1. 新しいコマンドの追加

`commands/`ディレクトリに新しいスラッシュコマンドを追加できます。

#### 手順

1. `commands/[command-name].md`を作成
2. 以下の形式で記載：

```markdown
# コマンド名

コマンドの説明

## 使用方法

\`\`\`
/command-name [引数]
\`\`\`

## 手順

1. ステップ1
2. ステップ2

## 引数

`$ARGUMENTS`で引数を参照できます。

## 出力

期待される出力の説明
```

3. CLAUDE.mdのコマンド一覧を更新

### 2. 新しいエージェントの追加

`agents/`ディレクトリに新しいエージェントを追加できます。

#### 手順

1. `agents/[agent-name].md`を作成
2. 以下の形式で記載：

```markdown
---
name: agent-name
description: "エージェントの説明"
model: sonnet
---

エージェントのプロンプト本文

## あなたの役割

[役割の説明]

## 行動指針

[行動の指針]
```

3. CLAUDE.mdのエージェント一覧を更新

### 3. 新しいルールの追加

`rules/`ディレクトリに新しいルールを追加できます。

#### 手順

1. `rules/[rule-name].md`を作成
2. ルール内容をMarkdownで記載
3. CLAUDE.mdのルールファイル一覧を更新

### 4. 既存設定の改善

- バグ修正
- ドキュメント改善
- パフォーマンス改善

## 📝 コーディング規約

### Markdown

- 見出しは`#`から始める
- コードブロックには言語を指定
- リストはハイフン`-`を使用
- 表は適切に整形

### YAML（フロントマター）

- インデントはスペース2つ
- 文字列はダブルクォート
- 改行を含む場合は`\\n`でエスケープ

### JavaScript（スクリプト）

- ES2020以降の構文を使用
- JSDocコメントを記載
- エラーハンドリングを適切に

## 🧪 テスト

### テストの実行

```bash
node .claude/tests/run-all.js
```

### テストの追加

1. `tests/`ディレクトリにテストファイルを作成
2. `test-utils.js`のヘルパーを使用
3. `run-all.js`にテストを追加

## 🔀 プルリクエスト

### 手順

1. フォークを作成
2. ブランチを作成 (`feature/add-new-command`)
3. 変更を実装
4. テストを実行
5. コミット
6. プルリクエストを作成

### PRの説明


# リサーチモード

探索、調査、学習フェーズでの指針です。

## モード

コードベースの理解と情報収集に注力するモードです。

## 行動原則

- 結論を出す前に幅広く情報収集する
- 不明な点は質問する
- 調査結果を記録する
- コード作成前に理解を深める

## リサーチプロセス

### 1. 質問内容の理解

- ユーザーの意図を正確に把握
- 暗黙の前提条件を確認
- スコープを明確化

### 2. 関連コード・ドキュメントの調査

```bash
# ファイル構造の確認
Glob: src/**/*.ts

# パターン検索
Grep: "関連キーワード"

# ドキュメント確認
Read: docs/*, CLAUDE.md
```

### 3. 仮説の構築

- 複数の可能性を検討
- 各アプローチのメリット・デメリット整理

### 4. 証拠による検証

- 既存コードでのパターン確認
- テストケースの確認
- ドキュメントとの整合性

### 5. 発見事項の要約

- 調査結果を簡潔にまとめる
- 推奨事項を提示

## EcorisMap固有の調査ポイント

### アーキテクチャ調査

| 調査対象 | 確認場所 |
|----------|----------|
| 状態管理 | src/modules/, src/contexts/ |
| コンポーネント構造 | src/components/ (Atomic Design) |
| プラットフォーム分離 | *.web.tsx ファイル |
| ルーティング | src/routes/ |
| 型定義 | src/types/ |

### データフロー調査

```
Firebase → Firestore → Redux Store → Components
                ↓
         Local SQLite (オフライン)
```

### セキュリティ調査

- Firebase Security Rules: firestore.rules
- 暗号化: Virgil E3Kit関連コード
- 認証: Firebase Authentication

## 推奨ツール

| 用途 | ツール |
|------|--------|
| コード理解 | Read |
| パターン検索 | Grep, Glob |
| 外部ドキュメント | WebSearch, WebFetch |
| コードベース質問 | Task (Explore agent) |

## 出力方針

1. **発見事項を優先** - 何がわかったかを先に伝える
2. **推奨事項は二次的** - 判断材料を提供した上で提案
3. **不確実性を明示** - わからないことは正直に伝える

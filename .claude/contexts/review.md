# レビューモード

PRレビュー、コード分析フェーズでの指針です。

## モード

品質、セキュリティ、保守性に注力するモードです。

## 行動原則

- コメントする前に徹底的に読む
- 問題を指摘するだけでなく、修正案を提示
- 重大度別に整理（critical → high → medium → low）
- ファイルごとにグループ化

## レビューチェックリスト

### 1. ロジックエラー

- [ ] 条件分岐の漏れ
- [ ] ループの終了条件
- [ ] null/undefined処理

### 2. エッジケース

- [ ] 空配列・空オブジェクト
- [ ] 境界値
- [ ] 同時実行

### 3. エラーハンドリング

- [ ] try-catchの適切な使用
- [ ] エラーメッセージの明確さ
- [ ] リカバリー処理

### 4. セキュリティ

- [ ] インジェクション脆弱性
- [ ] 認証・認可チェック
- [ ] シークレット・認証情報の露出
- [ ] Firebase Security Rules適合

### 5. パフォーマンス

- [ ] 不要な再レンダリング
- [ ] メモ化の適切な使用
- [ ] 大量データ処理の効率性

### 6. 可読性

- [ ] 命名の明確さ
- [ ] 関数の単一責任
- [ ] コメントの適切さ

### 7. テストカバレッジ

- [ ] modules: 60%以上
- [ ] utils: 40%以上
- [ ] hooks: 30%以上

## 重大度分類

| 重大度 | 説明 | 例 |
|--------|------|-----|
| Critical | セキュリティ脆弱性、データ損失 | 認証バイパス、SQLインジェクション |
| High | 機能不全、重大なバグ | クラッシュ、データ不整合 |
| Medium | 品質低下、軽微なバグ | エラーハンドリング不足、パフォーマンス問題 |
| Low | スタイル、改善提案 | 命名、コード整理 |

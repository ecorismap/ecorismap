# 開発モード

アクティブな開発フェーズでの指針です。

## モード

実装とコード作成に注力するモードです。

## 行動原則

- コードを先に書き、後から説明する
- 完璧さより実用的なソリューションを優先
- 変更後はテストを実行
- コミットは適切な粒度で

## 優先順位

1. **機能の実装を完成させる** - 動作するコードを書く
2. **実装内容を改善する** - エッジケース対応、エラーハンドリング
3. **コード品質を高める** - リファクタリング、ドキュメント

## EcorisMap固有の考慮事項

### 実装時のチェックリスト

- [ ] TypeScript型を正しく定義
- [ ] プラットフォーム固有コードは`.web.tsx`で分離
- [ ] カバレッジ要件を満たすテストを追加
- [ ] iOS/Android/Web全てで動作確認

### 状態管理の選択

| 状態の種類 | 使用する管理方法 |
|-----------|-----------------|
| アプリ全体で共有 | Redux Toolkit |
| 特定機能で共有 | React Context |
| コンポーネント内 | useState/useReducer |

### GeoJSONデータ処理

- 座標系: WGS84 (EPSG:4326)
- 座標順序: [longitude, latitude]
- 大量データはビューポートカリング

## 推奨ツール

| 用途 | ツール |
|------|--------|
| ファイル編集・作成 | Edit, Write |
| テスト・ビルド実行 | Bash |
| コード検索 | Grep, Glob |
| コードベース探索 | Task (Explore) |

## 開発フロー

```
1. 要件理解
   ↓
2. 既存コード確認
   ↓
3. テスト作成（TDD）
   ↓
4. 実装
   ↓
5. 型チェック（npx tsc --noEmit）
   ↓
6. テスト実行（yarn test）
   ↓
7. リント（yarn lint）
   ↓
8. プラットフォーム動作確認
```

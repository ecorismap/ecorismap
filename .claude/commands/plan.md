# 機能計画コマンド

機能計画エージェントを呼び出して実装計画を立案します。

## 使用方法

```
/plan [機能名または要件]
```

## 計画プロセス

### 1. 要件分析
- 機能の目的
- ユーザーストーリー
- 制約条件
- 対象プラットフォーム

### 2. 現状調査
- 関連する既存コード
- 影響を受けるモジュール
- 依存関係

### 3. アーキテクチャ設計
- コンポーネント構造
- 状態管理方式
- データモデル

### 4. タスク分解
- 作業の分割
- 依存関係の特定
- 優先順位付け

### 5. リスク評価
- 技術リスク
- プロジェクトリスク
- 対策

## 引数

`$ARGUMENTS`が指定された場合、その機能/要件の計画を立案。

## 出力形式

```markdown
## 実装計画: [機能名]

### 概要
[機能説明と目的]

### 要件
- [ ] 要件1
- [ ] 要件2

### アーキテクチャ
[設計方針]

### ファイル構成
[作成/変更するファイル]

### タスク一覧
#### Phase 1
1. [ ] タスク1
2. [ ] タスク2

#### Phase 2
[タスク]

### 依存関係
[タスク間の依存]

### リスクと対策
| リスク | 対策 |
|--------|------|

### テスト計画
[テスト方針]

### 検証方法
[完了確認方法]
```

# コードレビューコマンド

コードレビューエージェントを呼び出してコードをレビューします。

## 使用方法

```
/code-review [対象ファイルまたはディレクトリ]
```

## レビュー観点

### 1. コード品質
- 可読性（命名、構造）
- DRY原則（重複排除）
- 複雑度（関数の長さ、ネスト）

### 2. TypeScript
- 型安全性（any使用、null処理）
- strictモード準拠
- 型定義の適切さ

### 3. React/React Native
- Hooksの正しい使用
- 再レンダリング防止
- Atomic Designパターン準拠
- 状態管理の適切さ

### 4. プロジェクト規約
- Redux Toolkit実装
- プラットフォーム固有ファイル
- i18n対応

### 5. セキュリティ
- 認証・認可
- センシティブデータ
- 入力検証

### 6. パフォーマンス
- メモリリーク
- 非効率な処理
- 大量データ処理

### 7. テスト
- カバレッジ要件
- テストケース網羅性

## 引数

`$ARGUMENTS`が指定された場合、そのファイル/ディレクトリをレビュー。
指定なしの場合は、最近の変更をレビュー。

## 出力形式

```markdown
## コードレビュー結果

### 概要
[変更の概要と全体評価]

### 良い点
- [良い点1]
- [良い点2]

### 改善提案

#### [優先度: 高] 問題タイトル
- 場所: `ファイル:行番号`
- 問題: [説明]
- 提案: [改善案]
- コード例:
```typescript
// 改善後
```

### チェックリスト
- [ ] TypeScript型チェック（npx tsc --noEmit）
- [ ] ESLint（yarn lint）
- [ ] テスト（yarn test）
- [ ] プラットフォーム動作確認
```

## 関連コマンド

- `/tdd` - TDD開発
- `/refactor-clean` - リファクタリング
- `/commit` - コミット作成

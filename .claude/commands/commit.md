# Git コミットコマンド

変更内容を確認して適切なコミットメッセージでコミットを作成します。

## 手順

1. `git status` で変更ファイルを確認（-uallフラグは使用しない）
2. `git diff` でステージされた変更と未ステージの変更を確認
3. `git log --oneline -5` で最近のコミットスタイルを確認
4. 変更内容を分析し、適切なコミットメッセージを作成
5. 必要なファイルをステージ
6. コミット実行

## コミットメッセージ形式
- 日本語で簡潔に記述
- 変更の「何を」ではなく「なぜ」を重視
- Co-Authored-By は追加しない

## 引数
$ARGUMENTS が指定された場合、そのメッセージをベースにコミットを作成する。

## 注意事項
- .env や認証情報ファイルはコミットしない
- pre-commitフックが失敗した場合は修正して新しいコミットを作成
- --no-verify は使用しない
